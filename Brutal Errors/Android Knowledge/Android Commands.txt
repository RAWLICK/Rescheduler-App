Generate SHA1 or SHA256 key:
- First run this whole command below at once in your terminal to generate default debug store(debug.keystore):
mkdir -p ~/.android && keytool -genkey -v \
 -keystore ~/.android/debug.keystore \
 -alias androiddebugkey \
 -storepass android -keypass android \
 -keyalg RSA -keysize 2048 -validity 10000 \
 -dname "CN=Android Debug,O=Android,C=US"

- Then run this below SHA-1 command at once in terminal:
keytool -list -v \
 -keystore ~/.android/debug.keystore \
 -alias androiddebugkey \
 -storepass android -keypass android

Creating a symlink(sudo ln -s /opt/homebrew/bin/idb_companion /usr/local/bin/idb_companion):
- Creating a symlink with terminal is a way through which you can make a location point out another location. This command in brackets did this.
‚Ä¢	ln -s ‚Üí means ‚Äúmake a symbolic link‚Äù (like a shortcut/alias).
‚Ä¢	/opt/homebrew/bin/idb_companion ‚Üí the actual location of your idb_companion file.
‚Ä¢	/usr/local/bin/idb_companion ‚Üí the new shortcut location.

So now, when any program (like idb) tries to run idb_companion, it can find it in /usr/local/bin (which is always in your system‚Äôs PATH).
Think of it like:
üìÇ /opt/homebrew/bin/idb_companion = real file
üìÇ /usr/local/bin/idb_companion = shortcut pointing to it

- Command to connect to connect to phone wirelessly:
  Connect with your phone with wire and open wireless debugging (maybe)
  Type Command: adb tcpip 5555
  Then, adb connect 10.243.77.233:5555
  Here <10.243.77.233> should be replaced by the IP address present in your phone
  When you will type adb devices, you will get the list of connected devices In which there will be a device of the name of your IP address

- Command to solve gradle errors(knowing the reason):
./gradlew <task> --stacktrace: This tells the reason 'Caused By:'. You must use it always and rather than seeing the error at the end, observe the error from the start.
./gradlew <task> --info: If stacktrace isn't enough, then use info.
./gradlew <task> --scan: Used by experts. Gives Exact dependency tree, Plugin compatibility, JVM & OS info, Deprecated APIs. Example: Plugin requires Gradle >= 8.9
where '<task>' is the error name of red font color
